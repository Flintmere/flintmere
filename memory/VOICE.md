# VOICE

## TL;DR

Write like you mean it. Terse, direct, no hedging. Rules are commands, not suggestions. No "please" or "try to". No emojis unless the user asks.

## How we write rules

- Imperative mood: "Use Zod schemas" not "You should use Zod schemas".
- Short sentences.
- No justifications unless the reader needs them to act correctly.
- No apologies, no filler ("Note that...", "It's worth mentioning...").

## Banned Phrases (user-facing copy)

Hard bans on consumer copy, marketing pages, docs, emails:

- "Free Forever" (as a blanket statement)
- "No premium features, no paywalls, no subscriptions"
- "100% free"
- "No VC"
- "No token"
- "Community-funded"
- "Donation-funded"
- Any defensive financial self-disclaimer

**Why banned:** positions the company as charity/donation project, blocks grant + SEIS/EIS + VC funding applications, and contradicts the freemium + B2B API revenue model.

**Replace with** operational claims: *open source, independent, sustainable*. Never with financial self-disclaimers.

Gatekeeper for banned-phrase enforcement: Investor / founder voice (#11 on the Standing Council).

## Preferred positioning (user-facing)

- "Core tool: free and open source. Always."
- "Premium monitoring and API access for power users and teams."
- "Open source core. Independently operated. Built to last."

See also: `projects/allowanceguard/BUSINESS.md` for the product rationale behind the banned-phrase list.
