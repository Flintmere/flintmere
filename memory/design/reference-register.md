# reference-register.md — Design references for Flintmere

> **Standing Council ratification 2026-04-28.** This file is the canonical
> reference register every design-skill dispatch reads first. Replaces the
> implicit "Claude knows what good looks like" assumption with explicit
> named references. Operator's visual taste calibrates the register;
> Claude's brief reads it; output rises to the named reference instead of
> averaging to "competent neutral-bold AI aesthetic."
>
> **Two binding clauses:**
>
> 1. **Beauty earns its keep.** Decorative non-functional elements (dot-grids,
>    hairline ornaments, oversized index numerals, letter-cut shapes,
>    asymmetric ornaments, cropped silhouettes, deliberate misalignments)
>    are PERMITTED when they produce beauty. Beauty is a load-bearing
>    function. The earlier "every element earns its keep, no decoration"
>    framing is REPLACED by "decoration earns its keep BY being beautiful."
>    Per `tokens.md` §Standing rules updated 2026-04-28.
> 2. **Reference selection is council work, per dispatch.** Every design-
>    skill dispatch (`frontend-design`, `design-marketing-surface`,
>    `design-component`, `design-critique`, `design-motion`) starts with a
>    council pre-flight: name 3 references by URL with a one-sentence
>    annotation each on what to borrow. If the council can't name 3, the
>    surface isn't ready for design — return to grilling the requirement.

---

## How to use this file

- **Skills read this first** on every design dispatch. The relevant register
  entries inform the brief.
- **Operator strikes, adds, annotates** — the register is calibrated to
  operator's taste, not Claude's training.
- **Per-dispatch reference set** is a SUBSET of this register, picked by
  the council seat most relevant to the surface (e.g. #7 Maren picks for
  visual surfaces, #1 Editor picks for editorial-typography moments,
  #25 picks for imagery, #6 Idris picks for motion).
- **Never copy verbatim.** References inform aesthetic register, not
  layout. Borrow rhythms, not pixels.

## Section A — Editorial typography + heritage register

### Apartamento (apartamentomagazine.com)

**Borrow:** restraint as composition. Page after page of single image + one sentence + one credit line. The discipline of leaving things out. Use when a section is currently over-decorated.

## Section D — Decorative-as-load-bearing (operator's strategic shift, 2026-04-28)

These references are listed because they USE decoration well — proof that "decoration earns its keep when it produces beauty."

### Bureau Mirko Borsche project pages
**Borrow:** decorative ornament that punctuates rather than decorates. Hand-drawn arrows, oversized index numerals (`No. 03`), cropped silhouettes at the bleed.

### Pentagram case studies
**Borrow:** decorative typographic moments — a single oversized `&` or `/` as a section anchor. The character earns its keep because the page is unforgettable BECAUSE of it.

### Awwwards "Site of the Day" (awwwards.com/sotd) — quarterly review
**Borrow:** rotating reference set. Operator + Claude review one or two SOTDs per quarter, extract one decorative move per site, log here.

### Brutalist Web Design (brutalistwebsites.com)
**Borrow:** the anti-pattern catalogue (what NOT to do). Useful for `memory/design/anti-patterns.md` (separate doc, P2 follow-up).

### Anagrama (anagrama.com)
**Borrow:** Mexican design studio whose case-study pages use decoration as confidence — large-scale ornament that earns its keep by being unforgettable. Reference for any "this is the moment we need a flourish" decision.

### House Industries (houseind.com)
**Borrow:** type-foundry decoration — letterforms presented as the art object, not the typography. Reference for hero-class moments where the typography itself can BE the imagery.

### Tom Sachs Studio (tomsachs.org)

### Studio Output (studio-output.com)
**Borrow:** UK agency that uses oversized typography as the page's only ornament. Reference when a section is over-decorated and we're stripping back to type.

## Section F — Anti-references (NEVER borrow)

- **Stripe's gradient hero** — generic SaaS trope post-2022. We're past it.
- **Tailwind UI templates** — pattern-saturated; identifiable at 100m.
- **Linear's blue accent** — Linear's blue is theirs; ours is amber.
- **shadcn/ui defaults** — the shadows, the radii, the colour palette read as "AI generator did this." Built-from-scratch tokens beat shadcn-imported tokens for brand cohesion.
- **Any site with auto-playing video** — banned per `memory/design/motion.md`.
- **Any site that puts the wordmark inside a hero text block** — Flintmere wordmark `Flintmere]` lives in SiteHeader only. Hero text doesn't repeat the mark.

## Section G — Curation engines (rotating quality calibration, added 2026-04-28)

These are not specific references — they are *sources* the council pulls fresh references from on a rotating cadence. The pattern: operator + Claude review one or two SOTDs (Site of the Day) per source per month, extract one specific decorative or typographic move per site, log to whichever section §A–E fits best with attribution to the source.

The point of curation engines: stop our reference set from going stale. Fashion in design moves; what reads as confident in 2026-04 may read as dated by 2026-10. The register is a living document.

### Awwwards (awwwards.com)
**Use for:** Sites of the Day, Sites of the Month, Annual Award nominees. Strong on craft + interactive + editorial-typography moments. Heavy on motion sometimes — pull the typography + composition moves, leave the parallax-overload behind. **Cadence:** review monthly SOTDs, extract 1–2 references per quarter.

### The FWA — Favourite Website Awards (thefwa.com)
**Use for:** interactive + experimental + heavier-craft moments. Skews toward animation-rich and gimmick-prone — apply Idris's filter (motion is punctuation, not decoration). Best for: discovering one bold compositional move that survives the strip-back to Flintmere's calm canon. **Cadence:** review quarterly; extract sparingly (most FWA winners are too motion-loud for our register).

### CSS Design Awards (cssdesignawards.com)
**Use for:** CSS-craft-focused awards. More technically interesting than visually surprising. Strong on layout + typography + restrained motion. Closer to our register than thefwa on average. **Cadence:** review monthly featured site, extract 1–2 references per quarter.

### SiteInspire (siteinspire.com)
**Use for:** curated showcase that skews editorial + typographic. Less awards-driven than the three above; more "considered-design at scale". Filter by Style (e.g. "Minimal", "Typographic", "Photographic") to surface aligned references. **Cadence:** monthly browse, extract 1–2 references per quarter.

### Land-Book (land-book.com)
**Use for:** landing-page-specific curation. Most useful for `/pricing`, `/audit`, vertical landing pages, conversion-shaped surfaces. Skews SaaS-modern — apply the §F anti-reference filter to avoid Tailwind-template-saturation. **Cadence:** monthly check; high-vigilance against generic.

### Godly (godly.website)
**Use for:** Tailwind-skeptical + agency-led curation. Good counterweight to Land-Book — surfaces sites that look hand-crafted vs template-built. **Cadence:** quarterly browse.

### Httpster (httpster.net)
**Use for:** unfussy curation — single-page archive of well-designed sites without the awards-show theatrics. Lower volume but higher signal-to-noise for our register. **Cadence:** monthly check.

### Best Website Gallery (bestwebsite.gallery)
**Use for:** opinionated single-curator selection. Skews editorial + typographic. **Cadence:** quarterly browse.

### One Page Love (onepagelove.com)
**Use for:** single-page sites only — useful for the marketing-home, /audit landing-page, /score/[shop] surface lineage. **Cadence:** monthly check.

### Curation-engine protocol (binding)

When the council adds a reference from a curation source:
1. **Capture the URL of the specific site** (not just the SOTD-of-the-day URL — the actual site being awarded).
2. **Capture date observed** + **source** (e.g. "Awwwards SOTD 2026-04-15").
3. **Write the one-sentence borrow annotation** — what specifically transfers to Flintmere's register, not "looks great".
4. **Log to whichever section §A–E fits.** If it fits no section, the reference probably doesn't belong in Flintmere's register — pass.
5. **Per-dispatch lead seat** decides whether to elevate a curation-engine pull into the per-dispatch 3-reference set, or whether it stays as deep-bench.

If a site from a curation engine is loud, motion-heavy, gradient-led, or template-shaped — it goes into §F anti-references, not §A–E.

## Per-dispatch reference-selection process (binding from 2026-04-28)

When dispatching `frontend-design`, `design-marketing-surface`, `design-component`, `design-critique`, or `design-motion`:

1. **Council pre-flight (mandatory).** Name 3 references from this register, by URL, with a one-sentence annotation each on WHAT to borrow. The lead council seat for the surface (#7 Maren / #1 Editor / #25 image direction / #6 Idris / etc.) does this picking. Operator can override.
2. **Pass to skill** as a top-line section in the brief: *"References (binding): (1) URL — annotation. (2) URL — annotation. (3) URL — annotation."*
3. **Skill produces output** explicitly traced to the reference set: each major design move names which reference informed it.
4. **Critique gate post-build.** `design-critique` reviews against the named references. If output reads further from the register than the previous version, BLOCK and rebuild.

If the council can't name 3 references for a surface, the surface isn't ready for design dispatch — return to `grill-requirement` or `design-information-architecture` first.
